Problem
Customer support teams receive a large number of support tickets every day in text form.
These tickets describe different issues such as login problems, payment failures, bugs, or general queries.

Currently, tickets are:

1.Manually read and categorized
2.Manually prioritized
3.Often delayed due to human workload

This manual process is time‑consuming, inefficient, and prone to errors, especially when many tickets are repetitive in nature.

Solution
This project proposes a Python‑based Customer Support Ticket Classification and Auto‑Resolution System.

The system:
1. Accepts a customer support ticket as text input.
2.   Uses Machine Learning and NLP techniques to classify the ticket     into a predefined category

Predicts the priority of the ticket

Calculates a confidence score for the prediction

Uses rule‑based logic to decide whether the ticket can be auto‑resolved or should be forwarded to a human agent

Generates an appropriate response for auto‑resolvable tickets

This reduces manual effort and improves response efficiency.

Scope
Included in the project:
Python‑based text processing

Machine Learning classification using TF‑IDF

Priority prediction

Rule‑based decision logic

Python‑based user interaction (CLI / Streamlit)

Not included in the project:
Real‑time deployment

Large‑scale production systems

Deep learning models

Cloud infrastructure

The project focuses on core logic and academic learning objectives
